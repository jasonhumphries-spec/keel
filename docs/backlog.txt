[ ] Phase 2: Gmail Pub/Sub webhooks — background scanning
    Automatic scanning ~5min latency after email arrives, no user action needed.
    Steps:
    1. Create Pub/Sub topic keel-gmail-push in keel-6921a
    2. Grant gmail-api-push@system.gserviceaccount.com Publisher role
    3. Call Gmail watch() per user on sign-in (7-day expiry)
    4. Cloud Function push handler: receives notification, fetches thread, classifies
    5. Cloud Scheduler: weekly watch() renewal for all active users
