SECURITY / PRIVACY (V2 ROADMAP)
═══════════════════════════════════════════════

[ ] V2: Field-level encryption of sensitive Firestore fields
    Fields to encrypt: aiTitle, aiSummary, aiDetailedSummary, senderName,
    senderEmail, subject, signal descriptions, categoryHints.
    Fields to leave unencrypted (needed for server queries): status, categoryId,
    aiImportanceScore, receivedAt, signals.type/date/amount.
    Architecture: user generates key pair at onboarding. Public key stored in
    Firestore (unencrypted). Server encrypts results using public key before
    writing. Client decrypts on read using private key (browser memory only).
    Effort: ~6 sessions. Touches every Firestore read/write.
    Decision: implement after public launch. Current GCP AES-256 at-rest
    encryption is sufficient for alpha/beta. Priority increases once users pay
    and privacy positioning needs to be technically airtight.
