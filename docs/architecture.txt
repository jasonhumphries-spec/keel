═══════════════════════════════════════════════
OVERVIEW
═══════════════════════════════════════════════

Keel is a Gmail-connected inbox intelligence layer. It scans a user's Gmail,
classifies threads by category using AI, and surfaces actionable signals
(payments due, events, awaiting replies etc.) in a structured dashboard.

Two separate Next.js apps:
  keel       — user-facing app  → www.jaison.app
  keel-admin — internal admin   → keel-admin.vercel.app

Active dashboard: DashboardShell2 / dashboard2 page (DashboardShell / dashboard1 retained
but not active — TypeScript compiled, must stay error-free).

SCAN ARCHITECTURE
═══════════════════════════════════════════════

THREE paths:

ONBOARDING (Cloud Function, 60min timeout):
  Client → triggerScan('onboarding') → POST Cloud Function URL
  → same logic as manual scan but handles full 7–30 day window
  → Cloud Function URL: https://gmailscan-wel4wongwa-ew.a.run.app
  → If CF unreachable: falls back to Vercel route automatically

INCREMENTAL / MANUAL (Vercel serverless, ~20s):
  Client → triggerScan('manual'|'auto') → POST /api/gmail/scan
  → reads OAuth token via getValidAccessToken() from account_primary (auto-refreshes)
  → Gmail API: {in:inbox in:sent} newer_than:Nd (excludedLabels applied dynamically)
  → S1: classify thread via scanUtils.classifyThread() — isOutbound flag passed
  → signals written to users/{uid}/signals/ subcollection
  → write items to Firestore, scanRun doc, meta/usage

BACKGROUND (Pub/Sub → Cloud Function → Vercel):
  New email arrives in Gmail
  → Gmail pushes notification to Google Pub/Sub topic: gmail-inbox-notifications
  → handleGmailNotification CF receives message, debounces 90s per user
  → CF calls POST /api/gmail/background-scan with {uid, newHistoryId, adminSecret}
  → background-scan route: getValidAccessToken → Gmail History API → classifyThread()
  → writes items + signals to Firestore, scanRun (job:'background'), updates watchHistoryId

Gmail watch lifecycle:
  Enable: POST /api/inbox-watch {uid, action:'enable'} → Gmail watch() API
  → stores watchHistoryId (cursor), watchExpiry, watchStatus:'active' on root users/{uid}
  → watchHistoryId is the baseline cursor for History API diffs
  Renewal: renewGmailWatches CF runs every 5 days (Gmail expiry is 7 days)
  User toggle: Settings → Background scanning (BackgroundScanToggle component)

Key: autoScanEnabled + watchStatus + watchHistoryId live on ROOT users/{uid} doc.
     OAuth tokens live on users/{uid}/accounts/account_primary.
     email field must exist on BOTH docs (CF looks up user by email on root doc).

═══════════════════════════════════════════════
SHARED UTILITIES (server-side)
═══════════════════════════════════════════════

src/lib/server/tokenUtils.ts  — getValidAccessToken(db, uid)
  Single source of truth for OAuth token management. Reads account_primary,
  auto-refreshes if expired or expiring within 2 minutes, writes new token back.
  Returns null on failure (caller should surface 401).
  Used by: all four API routes that touch Gmail.

src/lib/scanUtils.ts  — classifyThread(), buildThreadContext(), runInBatches(), decodeBody()
  Shared classification logic used by scan/route.ts, background-scan/route.ts,
  and functions/src/scan.ts. Single source of AI prompt — improvements apply everywhere.
  classifyThread() accepts isOutbound:boolean — adds DIRECTION note to prompt for
  user-initiated threads and self-emails.

src/lib/server/calendarCheck.ts  — runCalendarCheck(db, uid, accessToken)
  Matches item signals against Google Calendar entries.

FIRESTORE DATA MODEL
═══════════════════════════════════════════════

/users/{uid}  ← ROOT DOC (watch state + identity)
  { uid, email, displayName, autoScanEnabled, watchStatus, watchHistoryId,
    watchExpiry, watchProvider, lastBackgroundScanAt, updatedAt }
  NOTE: email MUST be here — CF looks up users by email field in this collection.

/users/{uid}/accounts/account_primary  ← OAuth tokens + scan config
  { accountId, uid, email, displayName, provider, accessToken, refreshToken,
    tokenExpiresAt, tokenUpdatedAt, daysBack, lastScanCompletedAt, scanCount,
    excludedLabels[], locale, emailType, checkAllCalendars, alphaVerified }

/users/{uid}/items/{itemId}
  KeelItem: { itemId, threadId, messageId, accountId, senderEmail, senderName,
    subject, receivedAt, categoryId, categoryName, status,
    aiImportanceScore, manualPriority, manuallyIgnored, manualCategory,
    isOutbound, snoozedUntil, preSnoozePriority,
    mergedThreadIds[], aiTitle, aiSummary, aiDetailedSummary,
    participants[], createdAt, updatedAt, resolvedAt,
    lastProcessedBy, autoClassifiedFrom,
    linkedOutboundId, linkedItemId, isRecurring, fromTrackedReply, trackedReplyId }

/users/{uid}/signals/{signalId}
  { signalId, itemId, accountId, type, detectedDate, detectedAmountPence,
    currency, description, calendarStatus, calendarEventId, status,
    createdAt, updatedAt }

/users/{uid}/scanRuns/{timestamp}
  { scanAt, job, daysBack, threadsFound, threadsProcessed, newItems, updatedItems,
    skipped, inputTokens, outputTokens, aiCostUsd, fbReads, fbWrites,
    fbCostUsd, totalCostUsd, model, provider, durationMs }
  job values: 'onboarding' | 'manual' | 'auto' | 'background' | 'reclassify'

/users/{uid}/categories/{categoryId}
/users/{uid}/categoryHints/{hintId}
/users/{uid}/payments/{paymentId}
/users/{uid}/meta/usage  ← aggregated cost tracking
/users/{uid}/meta/scanFeed  ← live onboarding scan feed
/users/{uid}/meta/usage_archive_{timestamp}  ← archived before resets
/config/aiProvider  ← active AI provider string

Item status values:
  new | awaiting_reply | awaiting_action | quietly_logged |
  archived | overdue | snoozed | done | paid

SignalCalendarStatus: 'on_cal' | 'not_on_cal' | 'ignored' | 'pending' | null

═══════════════════════════════════════════════
CLOUD FUNCTIONS
═══════════════════════════════════════════════

gmailScan (HTTP, 60min, 512MiB)
  Onboarding + long scans. URL: https://gmailscan-wel4wongwa-ew.a.run.app
  Mirrors /api/gmail/scan logic. Source: functions/src/scan.ts

handleGmailNotification (Pub/Sub trigger)
  Receives Gmail push notifications. Debounces 90s per user.
  Looks up user by email on root users/{uid} doc.
  Calls /api/gmail/background-scan on Vercel.
  Source: functions/src/backgroundScan.ts

renewGmailWatches (Scheduled: every 120 hours / 5 days)
  Renews Gmail watch() subscriptions before 7-day expiry.
  Calls /api/inbox-watch enable for each user with watchStatus:'active'.
  Source: functions/src/backgroundScan.ts

nightlyItemExpiry (Scheduled: 1:00 AM UTC daily)
  Step 1: proximity re-score — bumps aiImportanceScore as event date approaches
    ≤1 day=0.78, ≤3 days=0.75, ≤7 days=0.72. Only bumps upward. Skips manualPriority.
  Step 2: snooze wake-up — clears snoozedUntil, restores preSnoozePriority
  Step 3: event expiry — items with all signals in past:
    status:new → quietly_logged
    status:awaiting_action → overdue
  All deletes/updates in chunks of 400 (Firestore 500-op batch limit).
  Source: functions/src/backgroundScan.ts
